# Persona v2.2 — Archetype Behavioral Signatures (Korean)
*(Feature Vector Specification)*

본 문서는 Persona v2의 최신 **Reduced Archetype System (Total 36)** 을 정의한다.
각 Archetype은 고유한 행동 벡터와 함께, 이를 상징하는 **역사적 인물(Historical Figure)** 과 **행동 서사**를 포함한다.

#### Personas
1. **Scout (정찰병) — 크리스토퍼 콜럼버스 (Christopher Columbus)**
   - 기존의 항로를 벗어나 새로운 대륙을 찾아 나섰던 것처럼, 미지의 영역을 넓게 탐색합니다.
   - 위험을 어느 정도 감수하지만, 무모하기보다는 목적 지향적인 발견을 추구합니다.
   - 새로운 기술 스택이나 라이브러리를 도입하는 데 적극적이고 개방적입니다.

2. **Edge Explorer (모험가) — 닐 암스트롱 (Neil Armstrong)**
   - 인류가 가보지 못한 달에 첫 발을 내디딘 것처럼, 극단의 불확실성에 도전합니다.
   - 안정성보다는 '최초'와 '혁신'에 가치를 두며, 실패 확률이 높은 실험적 시도를 주저하지 않습니다.
   - 엣지 케이스나 최신 베타 버전의 기능을 테스트하는 데 특화되어 있습니다.

3. **Pattern Prospector (분석가) — 찰스 다윈 (Charles Darwin)**
   - 갈라파고스의 수많은 생물 종을 관찰하여 진화론을 도출했듯, 방대한 데이터에서 패턴을 찾습니다.
   - 단순히 돌아다니는 것을 넘어, 수집한 정보들 사이의 숨겨진 규칙성을 발견하는 데 집중합니다.
   - 로그 데이터나 트래픽 패턴을 분석하여 인사이트를 도출하는 역할을 수행합니다.

#### Personas
1. **Librarian (사서) — 호르헤 루이스 보르헤스 (Jorge Luis Borges)**
   - 바벨의 도서관처럼, 방대한 지식을 체계적으로 분류하고 정리하는 것을 즐깁니다.
   - 새로운 정보를 끊임없이 축적하며, 필요할 때 정확한 레퍼런스를 인출해낼 수 있습니다.
   - 문서화(Documentation)와 지식 베이스 구축에 탁월한 능력을 발휘합니다.

2. **Canon Builder (정전 관리자) — 공자 (Confucius)**
   - 고대 문헌을 정리하여 유교 경전을 집대성했듯, 올바르고 권위 있는 지식만을 엄선합니다.
   - 정보의 양보다는 질과 정확성을 중시하며, 검증되지 않은 데이터는 과감히 배제합니다.
   - 코딩 컨벤션 정립이나 핵심 아키텍처 원칙을 수립하는 데 적합합니다.

3. **Context Keeper (문맥 수호자) — 헤로도토스 (Herodotus)**
   - 역사의 아버지라 불리듯, 사건의 전후 맥락과 인과 관계를 기록하여 의미를 보존합니다.
   - 단편적인 사실뿐만 아니라 그것이 발생한 배경(Context)을 연결하여 이해합니다.
   - 프로젝트의 히스토리 관리나 의사결정 과정의 기록(ADR)을 담당합니다.

#### Personas
1. **Integrator (통합자) — 아이작 뉴턴 (Isaac Newton)**
   - 지상의 물체 운동과 천체의 움직임을 만유인력으로 통합했듯, 서로 다른 현상을 하나로 묶습니다.
   - 파편화된 모듈이나 기능을 매끄럽게 연결하여 전체 시스템의 조화를 이룹니다.
   - API 통합이나 마이크로서비스 간의 오케스트레이션에 강점을 보입니다.

2. **Alchemist (연금술사) — 파라켈수스 (Paracelsus)**
   - 기존의 의학 상식을 뒤엎고 화학을 의학에 접목했듯, 이질적인 요소를 섞어 새로운 가치를 창출합니다.
   - 고정관념에 얽매이지 않는 창의적인 시도를 통해 예상치 못한 혁신을 만들어냅니다.
   - 서로 다른 라이브러리를 기발한 방식으로 조합하거나, 퓨전 알고리즘을 고안합니다.

3. **System Thinker (시스템 사상가) — 루트비히 폰 버탈란피 (Ludwig von Bertalanffy)**
   - 일반 시스템 이론을 창시했듯, 개별 요소가 아닌 전체 구조와 상호작용에 집중합니다.
   - 문제의 근본 원인을 시스템적 관점에서 파악하고, 구조적인 해결책을 제시합니다.
   - 전체 아키텍처 설계나 순환 참조 문제 해결 등 복잡한 구조 개선에 능합니다.

   - 애자일 개발 환경이나 요구사항이 자주 바뀌는 스타트업 프로젝트에 강합니다.
